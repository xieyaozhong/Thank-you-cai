# 謝謝你，菜！

「謝謝你，菜！」是一套給自家菜攤使用的雲端週報與接單系統。顧客可查看每週蔬果、依等級與產地篩選、加入菜籃並送出配送訂單；菜攤可同步庫存、安排出車、核銷到貨並發放水果點。

## 線上版本

- [GitHub Pages 展示版](https://xieyaozhong.github.io/Thank-you-cai/)：可直接瀏覽菜單、搜尋篩選、試玩菜籃、訂單預覽與抽卡收藏；資料只保存在使用者目前的瀏覽器。
- [正式雲端版](https://thank-you-cai.yao1230.chatgpt.site)：提供登入、共享庫存、真實訂單、配送核銷、水果點與攤主發布功能。

GitHub Pages 只能提供靜態檔案，不能執行 D1 資料庫與 `/api/*`。因此 `docs/` 是清楚標示用途的本機展示版，不會收集電話、地址或送出真實訂單。

## 完整功能

- 像素蔬果卡：16 種可編輯的 10×10 像素圖卡，涵蓋葉菜、根莖、瓜果與惜食品。
- 每週菜單：S／A／B／C 菜況規格、搜尋、分類、庫存與每單上限。
- 雲端訂單：伺服器重新驗價、計算運費、原子扣庫存並產生六位數核銷碼。
- 配送核銷：攤主安排出車，核銷後自動把點數發給下單帳號。
- 水果抽卡：每週一次免費抽卡，其後使用水果點；包含旋鈕、翻滾、落蛋與開蛋動畫，收藏與重複張數永久保存。
- 攤主更新：在草稿中調整等級、價格、庫存與菜況，再一次發布到所有裝置。
- 身分保護：使用 Sites／ChatGPT 提供的已驗證身分；第一位登入者成為攤主，其餘帳號為顧客。攤主操作皆在伺服器再次授權。
- 響應式與無障礙：支援桌機、平板與手機、鍵盤焦點、降低動態效果與語意化狀態提示。

## 資料設計

Cloudflare D1 是商品、使用者、訂單、明細、收藏與設定的唯一資料來源。瀏覽器只保存當下的介面狀態，不保存權威訂單或庫存。

- `users`：帳號、角色、點數與本週免費抽狀態
- `products`：已發布的商品、菜況、價格與庫存
- `orders`／`order_items`：訂單、配送、核銷與明細
- `collections`：每位使用者的抽卡收藏
- `settings`：本週區間與最後發布時間

資料表會由 `drizzle/` 內的 migration 建立；執行環境也會以冪等方式確認必要資料表與初始菜單存在。

## 本機開發

需求：Node.js 22.13 以上。

```bash
npm install
npm run db:generate
npm run dev
```

本機預覽會使用專案內的 D1 模擬環境，並提供固定的開發帳號。正式環境只信任平台注入的已驗證使用者標頭。

## 驗證

```bash
npm run lint
npm test
```

`npm test` 會先產生 Cloudflare Worker 相容的部署建置，再檢查完成版菜攤的伺服器輸出、網站資訊、持久化設定、主要 API，以及 GitHub Pages 靜態版的資源路徑與本機資料提示。

## 主要 API

- `GET /api/market`：菜單、目前帳號、訂單、點數與收藏
- `POST /api/orders`：建立訂單並扣庫存
- `POST /api/orders/:id/dispatch`：攤主安排出車
- `POST /api/orders/:id/verify`：攤主核銷並發點
- `PATCH /api/menu`：攤主發布本週菜單
- `POST /api/gacha`：抽卡並更新收藏／點數

部署由 `.openai/hosting.json` 宣告邏輯 D1 binding，實際資源與 migration 由 Sites 建立及套用。
